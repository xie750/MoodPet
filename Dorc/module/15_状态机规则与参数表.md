# 状态机规则与参数表

## 1. 文档目的

本文件用于约束情绪信号、用户状态、宠物状态、产品动作、冷却策略和提醒频率。

本项目不追求医疗级情绪判断，只判断可用于产品交互的轻量状态趋势。

## 2. 状态分层

系统状态分为三层：

```text
EmotionSignal 原始识别信号
-> UserState 用户状态
-> PetCommand / ProductAction 产品动作
```

约束：

1. 情绪识别模块只输出 `EmotionSignal`。
2. 状态决策引擎唯一负责计算 `UserState`。
3. 应用编排层负责把 `DecisionOutput` 转换成 `PetCommand`。
4. 桌宠模块只执行 `PetCommand`。

## 3. 输入信号

```ts
type EmotionSignal = {
  facePresent: boolean;
  smileScore: number;
  eyeClosedScore: number;
  headDownScore: number;
  activityScore: number;
  timestamp: number;
};
```

取值范围：

| 字段 | 范围 | 说明 |
| --- | --- | --- |
| facePresent | boolean | 是否检测到人脸 |
| smileScore | 0 到 1 | 微笑强度 |
| eyeClosedScore | 0 到 1 | 眼部闭合趋势 |
| headDownScore | 0 到 1 | 低头趋势 |
| activityScore | 0 到 1 | 表情或头部活动度 |
| timestamp | 毫秒时间戳 | 信号时间 |

所有 score 必须在进入状态机前裁剪到 0 到 1。

## 4. 用户状态

```ts
type UserState = "happy" | "calm" | "tired" | "away";
```

状态含义：

| 状态 | 含义 | 表达边界 |
| --- | --- | --- |
| happy | 用户有明显积极表情 | 只表达“看起来心情不错” |
| calm | 默认稳定状态 | 不评价情绪 |
| tired | 用户可能疲惫 | 只表达“可能有点累” |
| away | 用户离开摄像头 | 不推断原因 |

禁止输出：

1. sad
2. angry
3. depressed
4. anxious
5. stressed
6. 医疗或心理诊断类状态

## 5. 滑动窗口参数

默认采样频率：

| 场景 | 频率 |
| --- | --- |
| 普通陪伴 | 每 2 秒 1 次 |
| 游戏中 | 每 500ms 到 1 秒 1 次 |
| 专注模式 | 每 5 到 10 秒 1 次或暂停 |
| 摄像头关闭 | 不采样 |

滑动窗口：

| 参数 | 默认值 |
| --- | --- |
| 短窗口 | 30 秒 |
| 长窗口 | 180 秒 |
| 最小判定样本 | 5 个 |
| 状态切换最短持续 | 10 秒 |
| 状态回落缓冲 | 15 秒 |

说明：

1. 短窗口用于判断当前状态。
2. 长窗口用于判断疲惫持续和提醒冷却。
3. 样本不足时默认 `calm`。

## 6. 状态判定阈值

### 6.1 away

规则：

```text
facePresent = false 持续 120 秒
-> userState = away
```

快速离开提示：

```text
facePresent = false 持续 30 秒
-> 不切 away，仅降低宠物活跃度
```

恢复：

```text
facePresent = true 且连续 3 个样本有效
-> 退出 away
```

### 6.2 happy

规则：

```text
短窗口 smileScore 平均值 >= 0.65
且 facePresent = true
且持续 10 秒
-> userState = happy
```

增强条件：

```text
smileScore 峰值 >= 0.8
activityScore >= 0.4
```

退出：

```text
短窗口 smileScore 平均值 < 0.45 持续 15 秒
-> 回落到 calm
```

### 6.3 tired

规则一：

```text
短窗口 eyeClosedScore 平均值 >= 0.55
且持续 30 秒
-> userState = tired
```

规则二：

```text
短窗口 headDownScore 平均值 >= 0.55
且 activityScore <= 0.35
且持续 30 秒
-> userState = tired
```

规则三：

```text
长窗口内 activityScore 平均值 <= 0.25
且 smileScore 平均值 <= 0.3
且 facePresent = true
-> userState = tired
```

退出：

```text
smileScore 平均值 >= 0.5
或 activityScore 平均值 >= 0.45
持续 20 秒
-> 回落到 calm 或 happy
```

### 6.4 calm

默认状态。

进入条件：

```text
facePresent = true
且不满足 happy / tired
-> userState = calm
```

样本不足、置信度不足、规则冲突时统一回落到 `calm`。

## 7. 状态优先级

同一时间满足多个条件时，优先级如下：

```text
away > tired > happy > calm
```

但产品动作优先级独立计算：

```text
task_completed > game_finished > user_confirmed_action > tired_care > happy_feedback > idle
```

例子：

1. 用户完成任务时，即使当前 tired，也先庆祝，再温和提醒休息。
2. 游戏结束时，先反馈游戏结果，再根据状态回到 care / happy / idle。
3. away 时不主动发起游戏邀请。

## 8. 宠物状态映射

```ts
type PetState =
  | "idle"
  | "happy"
  | "care"
  | "tired"
  | "celebrate"
  | "sleep";
```

映射表：

| UserState / 事件 | PetState | 说明 |
| --- | --- | --- |
| calm | idle | 安静陪伴 |
| happy | happy | 正反馈 |
| tired | care | 温和关心 |
| away | sleep | 降低存在感 |
| task_completed | celebrate | 临时庆祝 |
| game_finished success | happy | 游戏成功反馈 |
| game_finished quit | care | 不责备，只安慰 |

临时状态持续时间：

| 状态 | 持续 |
| --- | --- |
| celebrate | 5 到 8 秒 |
| happy 强反馈 | 6 到 10 秒 |
| care 气泡 | 6 到 8 秒 |
| sleep | 直到检测到用户回来 |

## 9. ProductAction

```ts
type ProductAction =
  | "none"
  | "pet_idle"
  | "pet_happy"
  | "pet_care"
  | "pet_sleep"
  | "invite_smile_game"
  | "remind_break"
  | "celebrate_task";
```

动作生成规则：

| 条件 | ProductAction |
| --- | --- |
| calm | pet_idle |
| happy 首次进入 | pet_happy |
| tired 首次进入 | pet_care |
| tired 持续超过 180 秒 | invite_smile_game |
| away 持续超过 120 秒 | pet_sleep |
| task_completed | celebrate_task |

## 10. 冷却策略

冷却 key：

```text
tired_care_bubble
invite_smile_game
task_complete_celebrate
daily_all_done
happy_feedback
```

默认冷却：

| 动作 | quiet | normal | active |
| --- | --- | --- | --- |
| 疲惫关心气泡 | 60 分钟 | 30 分钟 | 15 分钟 |
| 邀请微笑小游戏 | 120 分钟 | 60 分钟 | 30 分钟 |
| 开心反馈 | 45 分钟 | 20 分钟 | 10 分钟 |
| 任务庆祝 | 不冷却 | 不冷却 | 不冷却 |
| 今日全部完成 | 每天 1 次 | 每天 1 次 | 每天 1 次 |

专注模式：

```text
focusMode = true
```

时：

1. 不显示主动气泡。
2. 不主动邀请小游戏。
3. 允许任务完成庆祝，但不播放强动效。
4. 宠物状态仍可安静变化。

## 11. 消息文案边界

疲惫类推荐表达：

```text
你可能有点累了，要不要先慢一点？
要不要休息 1 分钟？我在这里陪你。
我们先不急，做个很轻的小调整就好。
```

禁止表达：

```text
你状态很差。
你看起来很焦虑。
你是不是抑郁了。
你必须休息。
你今天效率太低。
```

## 12. 游戏难度参数

微笑能量小游戏：

| UserState | 目标能量 | 时长 | 能量倍率 |
| --- | --- | --- | --- |
| happy | 100 | 45 秒 | 1.2 |
| calm | 100 | 60 秒 | 1.0 |
| tired | 70 | 45 秒 | 1.1 |
| away | 暂停 | 暂停 | 0 |

能量增长：

```text
energyDeltaPerSecond = baseRate * (0.5 + smileScore) * multiplier
```

默认：

```text
baseRate = 1.5
```

识别中断：

```text
连续 5 秒无有效 smileScore
-> 使用默认增长 0.5 * baseRate
```

保证游戏可完成，不因识别失败惩罚用户。

## 13. 置信度计算

MVP 使用简单置信度：

```text
confidence = 有效样本比例 * 规则强度
```

有效样本比例：

```text
validSamples / expectedSamples
```

规则强度示例：

```text
happyStrength = clamp((avgSmile - 0.45) / 0.35, 0, 1)
tiredStrength = max(
  clamp((avgEyeClosed - 0.35) / 0.35, 0, 1),
  clamp((avgHeadDown - 0.35) / 0.35, 0, 1)
)
```

置信度低于 `0.45` 时：

```text
userState = calm
```

## 15. 开发验收

状态机模块完成后必须满足：

1. 能输入 mock `EmotionSignal` 序列。
2. 能稳定输出 `UserState`。
3. 样本不足时回落 `calm`。
4. 冷却期内不重复提醒。
5. `focusMode` 下不主动打扰。
6. `task_completed` 优先触发庆祝。
7. `away` 时不邀请游戏。
8. 单元测试覆盖 happy、calm、tired、away、冷却和优先级。
