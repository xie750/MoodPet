# 模块接口契约与事件总线设计文档

## 1. 文档目的

本文件用于约束所有模块的协作方式，避免多人分模块开发时出现职责冲突。

所有模块开发前必须先阅读本文件。模块文档如果与本文件冲突，以本文件为准。

## 2. 总体原则

1. 模块高内聚：每个模块只负责自己的领域能力。
2. 模块低耦合：模块之间不直接互相调用内部实现。
3. 共享契约统一：跨模块通信只使用本文件定义的类型、事件和接口。
4. 单一写入方：同一类状态只能由一个模块负责写入。
5. 副作用外置：状态决策引擎和规则模块只返回结果，不直接操作 UI、窗口或数据库。

## 3. 模块职责边界

| 模块 | 可以做 | 不可以做 |
| --- | --- | --- |
| 应用壳 | 窗口、托盘、IPC 注册 | 产品规则、数据库表设计 |
| 桌宠体验 | 渲染宠物、执行 PetCommand | 判断用户情绪、直接查任务 |
| 状态决策引擎 | 计算 UserState、DecisionOutput | 操作 UI、写数据库、开窗口 |
| 情绪识别 | 输出 EmotionSignal | 输出宠物动作、保存人脸图像 |
| 待办任务 | 任务 CRUD、TaskSummary、任务事件 | 直接修改宠物状态 |
| 微笑小游戏 | 游戏流程、GameRecord、游戏事件 | 直接调用宠物模块 |
| 对话陪伴 | 生成回复和推荐动作 | 直接执行推荐动作 |
| 设置隐私 | 保存设置、发布设置变化 | 直接控制其他模块内部逻辑 |
| 本地存储 IPC | 数据持久化、安全 IPC | 产品规则判断 |
| 视觉文案 | 资源和文案规范 | 业务状态判断 |

## 4. 应用编排层

应用编排层用于连接模块，但它不是新的业务模块。

建议目录：

```text
src/app/orchestrator/
```

它可以做：

1. 订阅事件总线。
2. 调用状态决策引擎。
3. 把 `DecisionOutput` 转换成 `PetCommand` 或 `NavigationCommand`。
4. 调用公开 IPC 接口保存必要数据。

它不可以做：

1. 自己计算情绪识别结果。
2. 自己实现任务规则。
3. 自己渲染 UI。
4. 自己定义数据库表。
5. 跳过模块接口直接修改其他模块内部状态。

## 5. 共享类型

```ts
type UserState = "happy" | "calm" | "tired" | "away";

type PetState =
  | "idle"
  | "happy"
  | "care"
  | "tired"
  | "celebrate"
  | "sleep";

type EmotionSignal = {
  facePresent: boolean;
  smileScore: number;
  eyeClosedScore: number;
  headDownScore: number;
  activityScore: number;
  timestamp: number;
};

type TaskSummary = {
  totalToday: number;
  unfinished: number;
  overdue: number;
  completedToday: number;
};

type AppSettings = {
  cameraEnabled: boolean;
  emotionEnabled: boolean;
  reminderLevel: "quiet" | "normal" | "active";
  focusMode: boolean;
  petAlwaysOnTop: boolean;
  launchAtStartup: boolean;
};

type PanelRoute = "tasks" | "chat" | "game" | "settings";
```

## 6. 命令类型

### 6.1 PetCommand

```ts
type PetCommand =
  | {
      type: "set_state";
      state: PetState;
      reason?: string;
    }
  | {
      type: "show_bubble";
      message: string;
      reason?: string;
      cooldownKey?: string;
    }
  | {
      type: "add_affinity";
      amount: number;
      reason?: string;
    }
  | {
      type: "add_energy";
      amount: number;
      reason?: string;
    };
```

只有桌宠模块和宠物命令处理方执行 `PetCommand`。其他模块只能发布事件或返回推荐。

### 6.2 NavigationCommand

```ts
type NavigationCommand = {
  type: "open_panel";
  route: PanelRoute;
};
```

只有应用壳或主面板路由层执行窗口/页面跳转。

## 7. 事件类型

```ts
type AppEvent =
  | EmotionSignalEvent
  | UserStateChangedEvent
  | TaskCreatedEvent
  | TaskCompletedEvent
  | GameStartedEvent
  | GameFinishedEvent
  | ChatRecommendedActionEvent
  | SettingsChangedEvent;
```

### 7.1 情绪事件

```ts
type EmotionSignalEvent = {
  type: "emotion_signal";
  payload: EmotionSignal;
  createdAt: number;
};

type UserStateChangedEvent = {
  type: "user_state_changed";
  payload: {
    userState: UserState;
    confidence: number;
    duration: number;
  };
  createdAt: number;
};
```

情绪识别模块只发布 `emotion_signal`。`user_state_changed` 由状态决策引擎发布。

### 7.2 任务事件

```ts
type TaskCreatedEvent = {
  type: "task_created";
  payload: {
    taskId: string;
    title: string;
  };
  createdAt: number;
};

type TaskCompletedEvent = {
  type: "task_completed";
  payload: {
    taskId: string;
    completedToday: number;
  };
  createdAt: number;
};
```

任务模块只发布任务事件，不直接修改宠物。

### 7.3 游戏事件

```ts
type GameStartedEvent = {
  type: "game_started";
  payload: {
    gameType: "smile_energy";
  };
  createdAt: number;
};

type GameFinishedEvent = {
  type: "game_finished";
  payload: {
    gameType: "smile_energy";
    result: "success" | "normal" | "quit";
    score: number;
    duration: number;
  };
  createdAt: number;
};
```

游戏模块只发布游戏事件，不直接修改宠物。

### 7.4 对话事件

```ts
type ChatRecommendedActionEvent = {
  type: "chat_recommended_action";
  payload: {
    action: "open_tasks" | "open_smile_game" | "take_break" | "none";
    userConfirmed: boolean;
  };
  createdAt: number;
};
```

对话模块只返回推荐动作。只有用户确认后，UI 才发布该事件。

### 7.5 设置事件

```ts
type SettingsChangedEvent = {
  type: "settings_changed";
  payload: Partial<AppSettings>;
  createdAt: number;
};
```

设置模块发布设置变化，其他模块自行响应。

## 8. 单一写入方规则

| 数据/状态 | 唯一写入方 |
| --- | --- |
| 任务数据 | 待办任务模块 |
| 游戏记录 | 小游戏模块 |
| 设置数据 | 设置与隐私模块 |
| 原始 EmotionSignal | 情绪识别模块，仅内存 |
| UserState | 状态决策引擎 |
| 宠物运行态 | 桌宠体验模块 |
| 宠物亲密度/能量 | 宠物命令处理方 |
| 窗口状态 | 应用壳模块 |

## 9. 推荐事件流

### 9.1 情绪驱动宠物

```text
情绪识别模块发布 emotion_signal
-> 状态决策引擎计算 UserState 和 DecisionOutput
-> 应用编排层生成 PetCommand
-> 桌宠模块执行 PetCommand
```

### 9.2 任务完成庆祝

```text
待办模块完成任务
-> 发布 task_completed
-> 应用编排层生成 set_state: celebrate 和 add_affinity
-> 桌宠模块执行状态变化
-> 本地存储保存宠物成长数据
```

### 9.3 游戏结束反馈

```text
小游戏模块写入 game_records
-> 发布 game_finished
-> 应用编排层生成 PetCommand
-> 桌宠模块执行反馈
```

### 9.4 对话推荐动作

```text
对话模块返回 recommendedAction
-> 用户点击确认
-> UI 发布 chat_recommended_action
-> 应用壳执行 NavigationCommand
```

## 10. 并行开发验收

每个模块交付时必须满足：

1. 不 import 其他业务模块的内部文件。
2. 只依赖共享类型和公开接口。
3. 不写入其他模块拥有的数据。
4. 不绕过事件总线直接触发其他模块 UI。
5. 能用 mock 输入单独运行或测试。
