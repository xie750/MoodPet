# 设置与隐私模块设计文档

## 1. 模块定位

设置与隐私模块负责建立用户信任。

对于摄像头类产品，隐私体验不是附属功能，而是产品能否被接受的前提。

## 2. MVP 设置项

| 设置项 | 类型 | 默认值 |
| --- | --- | --- |
| 摄像头开关 | boolean | false，首次授权后可开启 |
| 情绪识别开关 | boolean | true |
| 提醒等级 | quiet / normal / active | normal |
| 专注模式 | boolean | false |
| 宠物置顶 | boolean | true |
| 开机自启动 | boolean | false |

## 3. 首次使用隐私流程

```text
打开应用
-> 选择开始使用
-> 说明摄像头只做本地识别
-> 用户主动点击授权
-> 授权成功后开启识别
-> 设置页可随时关闭
```

不要在用户不理解的情况下自动打开摄像头。

## 4. 摄像头状态展示

产品需要明确告诉用户当前状态：

| 状态 | 展示 |
| --- | --- |
| 摄像头关闭 | 设置页显示关闭 |
| 摄像头运行中 | 宠物菜单或设置页显示运行中 |
| 识别暂停 | 显示暂停 |
| 加载失败 | 显示暂不可用 |

## 5. 数据存储说明

默认保存：

1. 任务数据。
2. 宠物亲密度。
3. 游戏记录。
4. 设置项。
5. 情绪状态摘要。

默认不保存：

1. 摄像头视频。
2. 人脸图片。
3. 单帧截图。
4. 可识别身份的图像数据。

## 6. 提醒等级

| 等级 | 行为 |
| --- | --- |
| quiet | 极少主动提醒，只保留状态变化 |
| normal | 默认低打扰提醒 |
| active | 更积极地提醒休息和小游戏 |

专注模式开启后，主动提醒显著减少，只保留用户主动交互和任务完成反馈。

## 7. 设置接口

```ts
type AppSettings = {
  cameraEnabled: boolean;
  emotionEnabled: boolean;
  reminderLevel: "quiet" | "normal" | "active";
  focusMode: boolean;
  petAlwaysOnTop: boolean;
  launchAtStartup: boolean;
};
```

IPC：

```ts
window.appApi.settings.get()
window.appApi.settings.update(patch)
```

设置模块只保存和展示用户偏好。其他模块通过订阅设置变更或读取 `AppSettings` 决定自己的行为。

例如：

1. 情绪识别模块根据 `cameraEnabled` 启停摄像头。
2. 状态决策引擎根据 `reminderLevel` 和 `focusMode` 调整提醒。
3. 应用壳根据 `petAlwaysOnTop` 调整窗口置顶。

设置模块不直接调用这些模块内部方法。

## 8. 模块边界

设置模块可以做：

1. 读取设置。
2. 更新设置。
3. 展示隐私说明。
4. 发布 `settings_changed` 事件。

设置模块不可以做：

1. 直接开启摄像头流。
2. 直接修改桌宠窗口。
3. 直接改变状态决策规则。
4. 直接清空其他模块数据，除非有明确数据管理接口。

## 9. 验收标准

1. 首次使用不会自动打开摄像头。
2. 用户可开启/关闭摄像头。
3. 摄像头关闭后情绪识别停止。
4. 用户可设置提醒等级。
5. 专注模式能抑制主动邀请。
6. 设置重启后仍然保留。
7. 文案明确说明不保存人脸图像。
