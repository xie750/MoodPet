# 状态决策引擎模块设计文档

## 1. 模块定位

状态决策引擎是产品中枢。

它负责把情绪识别、任务状态、用户设置和提醒冷却转换成具体产品动作。该模块必须独立于 UI，不能把规则散落在宠物组件、任务组件或游戏组件里。

该模块只做纯逻辑计算，不直接操作 React 组件、Electron 窗口、SQLite 或摄像头。

## 2. 三层状态模型

```text
EmotionSignal 原始识别信号
-> UserState 用户状态
-> ProductAction 产品动作
```

## 3. 输入类型

```ts
type EmotionSignal = {
  facePresent: boolean;
  smileScore: number;
  eyeClosedScore: number;
  headDownScore: number;
  activityScore: number;
  timestamp: number;
};

type UserState = "happy" | "calm" | "tired" | "away";

type TaskSummary = {
  totalToday: number;
  unfinished: number;
  overdue: number;
  completedToday: number;
};

type DecisionSettings = {
  cameraEnabled: boolean;
  emotionEnabled: boolean;
  reminderLevel: "quiet" | "normal" | "active";
  focusMode: boolean;
};
```

## 4. 输出类型

```ts
type PetState =
  | "idle"
  | "happy"
  | "care"
  | "tired"
  | "celebrate"
  | "sleep";

type ProductAction =
  | "none"
  | "pet_idle"
  | "pet_happy"
  | "pet_care"
  | "pet_sleep"
  | "invite_smile_game"
  | "remind_break"
  | "celebrate_task";

type NavigationAction = "none" | "open_tasks" | "open_chat" | "open_smile_game" | "open_settings";

type PetCommand = {
  type: "set_state" | "show_bubble";
  state?: PetState;
  message?: string;
  reason?: string;
};

type DecisionOutput = {
  userState: UserState;
  confidence: number;
  petState: PetState;
  action: ProductAction;
  petCommand?: PetCommand;
  navigationAction?: NavigationAction;
  message?: string;
  cooldownKey?: string;
};
```

## 5. MVP 判断规则

| 条件 | UserState | PetState | Action |
| --- | --- | --- | --- |
| 2 分钟未检测到人脸 | away | sleep | pet_sleep |
| 30 秒 smileScore 均值较高 | happy | happy | pet_happy |
| 30 秒 eyeClosed/headDown 较高 | tired | care | pet_care |
| tired 持续 3 分钟且不在冷却 | tired | care | invite_smile_game |
| 正常有人脸且无明显异常 | calm | idle | none |

## 6. 冷却策略

必须防止宠物过度打扰。

建议：

| 动作 | normal 冷却 | quiet 冷却 | active 冷却 |
| --- | --- | --- | --- |
| remind_break | 45 分钟 | 90 分钟 | 25 分钟 |
| invite_smile_game | 60 分钟 | 120 分钟 | 40 分钟 |
| task_overdue | 30 分钟 | 60 分钟 | 15 分钟 |

focusMode 开启时，只允许：

1. 宠物状态变化。
2. 任务完成庆祝。
3. 用户主动点击后的反馈。

不主动邀请游戏。

## 7. 设计原则

1. 不输出医疗化结论。
2. 不把单帧表情当成真实情绪。
3. 不因低置信度触发强动作。
4. 不重复催促用户。
5. 用户关闭摄像头后不推断情绪。

## 8. 模块边界

状态决策引擎可以做：

1. 根据 `EmotionSignal` 计算 `UserState`。
2. 根据任务摘要、设置和冷却信息生成 `DecisionOutput`。
3. 输出 `PetCommand` 和推荐动作。

状态决策引擎不可以做：

1. 直接调用 `pet.updateState`。
2. 直接打开窗口。
3. 直接写入 SQLite。
4. 直接请求摄像头。
5. 直接读取 React 组件状态。

## 9. 开发建议

第一版可以先用模拟信号测试：

```ts
const tiredSignal = {
  facePresent: true,
  smileScore: 0.1,
  eyeClosedScore: 0.7,
  headDownScore: 0.6,
  activityScore: 0.2,
  timestamp: Date.now(),
};
```

等规则跑通后再接真实 MediaPipe。

## 10. 验收标准

1. 输入 happy 信号，输出 petState = happy。
2. 输入 tired 持续信号，输出 petState = care。
3. tired 超过阈值后输出 invite_smile_game。
4. 冷却期内不重复 invite_smile_game。
5. cameraEnabled=false 时输出 calm/idle，不进行情绪推断。
6. away 状态能稳定触发 sleep。
7. 输出结果是纯对象，不产生 UI、数据库或窗口副作用。
