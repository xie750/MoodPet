# 对话陪伴模块设计文档

## 1. 模块定位

对话陪伴模块不是通用 AI 聊天机器人。

MVP 阶段它的目标是让宠物能基于当前情绪、任务和游戏上下文，给出短、温和、可执行的陪伴反馈。

## 2. MVP 策略

先使用规则回复，不优先接大模型。

原因：

1. 可离线演示。
2. 回复风格可控。
3. 避免医疗和安全风险。
4. 实现成本低。
5. 足够支撑 MVP 闭环。

## 3. 输入上下文

```ts
type ChatContext = {
  message: string;
  userState: "happy" | "calm" | "tired" | "away";
  taskSummary: {
    totalToday: number;
    unfinished: number;
    completedToday: number;
    overdue: number;
  };
  lastGameResult?: "success" | "normal" | "quit";
  petAffinity: number;
};
```

## 4. 输出结构

```ts
type ChatRecommendedAction = "open_tasks" | "open_smile_game" | "take_break" | "none";

type ChatReply = {
  reply: string;
  recommendedAction?: ChatRecommendedAction;
};
```

对话模块只返回推荐动作，不直接打开窗口、不直接启动游戏、不直接修改任务或宠物状态。UI 或应用编排层负责执行用户确认后的动作。

## 5. 规则分类

| 用户表达 | 识别方式 | 回复策略 |
| --- | --- | --- |
| 累、困、没精神 | 关键词 + tired | 建议休息或小游戏 |
| 任务好多、做不完 | 关键词 + unfinished | 建议先做最小任务 |
| 想玩、放松 | 关键词 | 推荐微笑能量游戏 |
| 不想做了 | 关键词 | 接纳情绪，降低压力 |
| 做完了、完成了 | 关键词 + taskSummary | 庆祝 |

## 6. 回复风格

必须做到：

1. 简短。
2. 温和。
3. 不说教。
4. 不诊断。
5. 不制造压力。
6. 更像伙伴，不像老师或医生。

避免：

```text
你应该提高效率。
你现在很焦虑。
你必须完成任务。
你可能有心理问题。
```

推荐：

```text
那我们先慢一点吧。我陪你缓一会儿。
```

## 7. 示例回复

用户：我有点累。

```json
{
  "reply": "那我们先慢一点吧。要不要玩 1 分钟轻松小游戏，我陪你缓一下。",
  "recommendedAction": "open_smile_game"
}
```

用户：今天任务好多。

```json
{
  "reply": "我们先挑一个最小的开始。完成一个也算前进，我陪你。",
  "recommendedAction": "open_tasks"
}
```

用户：我不想做了。

```json
{
  "reply": "可以先停一下。你不是失败了，只是需要换口气。",
  "recommendedAction": "take_break"
}
```

## 8. 安全边界

如果用户表达强烈自伤、伤害他人或极端负面内容，MVP 不做心理咨询，只做温和提示：

```text
听起来你现在真的很难受。请先联系身边可信任的人，或寻求当地紧急帮助。
```

不要给医疗建议，不要做诊断。

## 9. 模块边界

对话模块可以做：

1. 根据上下文生成规则回复。
2. 输出推荐动作。
3. 标记高风险表达并返回安全提示。

对话模块不可以做：

1. 直接打开任务页或小游戏。
2. 直接修改任务。
3. 直接修改宠物状态。
4. 直接读取摄像头。

## 10. 验收标准

1. 用户可以打开聊天窗口。
2. 输入文本后能得到回复。
3. 回复能结合 userState 和 taskSummary。
4. “我累了”能推荐小游戏或休息。
5. “任务好多”能推荐今日任务。
6. 回复不出现诊断性表达。
7. 大模型 API 不开启时仍可正常使用。
8. recommendedAction 只是建议，不产生副作用。
