# 视觉资源与文案模块设计文档

## 1. 模块定位

视觉资源与文案模块决定产品是否有陪伴感。

即使技术闭环成立，如果宠物不可爱、文案像系统通知、提醒太硬，产品也会失败。

## 2. 宠物视觉原则

MVP 只需要一个宠物，但状态表现要清楚。

原则：

1. 状态少，但每个状态明显。
2. 动作幅度温和，不影响工作。
3. 避免高频闪烁。
4. 角色要有“陪伴感”，不是工具图标。
5. 资源优先稳定，其次精美。

## 3. 必需资源

| 资源 | 优先级 | 用途 |
| --- | --- | --- |
| idle | P0 | 默认陪伴 |
| care | P0 | 疲惫关心 |
| happy | P0 | 积极反馈 |
| celebrate | P0 | 任务完成 |
| sleep | P0 | 用户离开 |
| tired | P1 | 慢节奏陪伴 |

每个状态可以先用：

1. WebP 动图。
2. PNG 序列帧。
3. Lottie。
4. CSS 占位动画。

## 4. 文案风格

关键词：

```text
短
温和
低压力
不诊断
不命令
像伙伴
```

## 5. 推荐文案库

### 5.1 疲惫提醒

```text
感觉你坚持很久啦，要不要缓 1 分钟？
我们先慢一点，我陪你。
要不要喝口水，换口气？
```

### 5.2 游戏邀请

```text
要不要陪我充 1 分钟能量？
来玩一小会儿，轻轻松松的。
我有点没电啦，可以帮我充点能量吗？
```

### 5.3 任务完成

```text
完成一个啦，真不错。
今天又前进了一点。
这个任务收下啦，我们继续慢慢来。
```

### 5.4 任务很多

```text
我们先挑一个最小的开始。
不用一次做完，先走一步就好。
任务有点多，我们慢慢拆。
```

### 5.5 用户离开

```text
我先在这里等你。
你回来啦。
```

## 6. 禁用表达

不要使用：

```text
你焦虑了。
你抑郁了。
你状态很差。
你必须完成。
你太拖延了。
你应该更努力。
```

这些表达会制造压力，也不符合产品边界。

## 7. UI 气质

建议视觉方向：

1. 桌宠轻量、柔和、但不要幼稚到影响办公。
2. 面板像效率工具，清爽克制。
3. 小游戏可以更活泼。
4. 设置页要可信、清晰。
5. 隐私说明要直白，不堆概念。

## 8. 验收标准

1. 每个宠物状态能被用户一眼区分。
2. 文案不出现医疗诊断词。
3. 提醒不命令用户。
4. 气泡文案长度适合桌面展示。
5. 任务完成反馈有正向情绪。
6. 疲惫提醒不会让用户有负担。
