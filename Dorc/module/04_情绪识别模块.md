# 情绪识别模块设计文档

## 1. 模块定位

情绪识别模块负责本地摄像头采集和面部特征提取。

注意：本模块不负责判断用户真实心理状态，也不输出“抑郁”“焦虑”等医疗判断。MVP 只输出可用于产品交互的基础状态信号。

## 2. MVP 目标

1. 请求摄像头授权。
2. 显示摄像头工作状态。
3. 使用 MediaDevices 获取视频流。
4. 使用 MediaPipe Face Landmarker 提取面部特征。
5. 输出标准化 EmotionSignal。
6. 不保存原始图片和视频。

## 3. 输出信号

```ts
type EmotionSignal = {
  facePresent: boolean;
  smileScore: number;
  eyeClosedScore: number;
  headDownScore: number;
  activityScore: number;
  timestamp: number;
};
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| facePresent | 是否检测到人脸 |
| smileScore | 微笑程度 |
| eyeClosedScore | 眼部闭合程度 |
| headDownScore | 低头/头部下垂程度 |
| activityScore | 表情或头部活跃度 |
| timestamp | 采样时间 |

## 4. 采样策略

| 场景 | 采样频率 |
| --- | --- |
| 常规桌宠陪伴 | 每 2 秒一次 |
| 小游戏中 | 每 0.5 到 1 秒一次 |
| 用户离开超过 2 分钟 | 降低到每 5 秒一次 |
| 专注模式 | 降低频率或暂停主动提醒 |
| 摄像头关闭 | 不采样 |

## 5. 用户状态输出

情绪识别模块在 MVP 中只输出 `EmotionSignal`。

`UserState` 由状态决策引擎统一计算，避免情绪识别模块和状态决策引擎同时判断用户状态导致冲突。

共享契约中保留的用户状态只有：

```ts
type UserState = "happy" | "calm" | "tired" | "away";
```

情绪识别模块不输出：

1. sad。
2. angry。
3. anxious。
4. depressed。
5. stressed。

这些容易误判，也容易造成用户反感。

## 6. 对外接口

```ts
type EmotionProvider = {
  start(): Promise<void>;
  stop(): Promise<void>;
  setSamplingMode(mode: "normal" | "game" | "low_power"): void;
  getLatestSignal(): EmotionSignal | null;
  subscribe(listener: (signal: EmotionSignal) => void): () => void;
};
```

小游戏可以通过 `getLatestSignal()` 或订阅方式读取 `smileScore`，但不能直接控制摄像头生命周期。

## 7. 隐私要求

必须做到：

1. 用户主动授权后才开启摄像头。
2. 设置页可随时关闭摄像头。
3. 宠物或设置页展示摄像头状态。
4. 默认不保存原始图片。
5. 默认不保存视频流。
6. 不上传人脸图像。
7. 只保存状态摘要，例如 tired 持续时间。

## 8. 异常处理

| 异常 | 处理 |
| --- | --- |
| 用户拒绝摄像头 | 进入普通桌宠模式 |
| 摄像头被占用 | 提示识别暂不可用 |
| MediaPipe 加载失败 | 回退模拟状态或 calm |
| 无人脸 | 输出 facePresent=false |
| 低置信度 | 默认 calm |

## 9. 验收标准

1. 用户未授权时不打开摄像头。
2. 授权后可以获取视频流。
3. 检测到人脸时 facePresent=true。
4. 用户微笑时 smileScore 有明显上升。
5. 用户离开时 facePresent=false。
6. 不生成任何人脸图片文件。
7. 摄像头关闭后识别完全停止。
8. 不输出 UserState，只输出 EmotionSignal。
