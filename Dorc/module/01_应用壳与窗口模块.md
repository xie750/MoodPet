# 应用壳与窗口模块设计文档

## 1. 模块定位

应用壳与窗口模块负责把产品变成一个真正的桌面应用。

该模块不负责宠物具体动画、不负责情绪识别、不负责小游戏逻辑。它只负责 Electron 应用生命周期、窗口创建、窗口通信、系统托盘和基础安全能力。

## 2. MVP 目标

MVP 阶段需要支持：

1. 启动 Electron 应用。
2. 创建透明置顶的宠物窗口。
3. 创建普通主面板窗口。
4. 支持宠物窗口拖动。
5. 支持点击宠物打开快捷菜单或主面板。
6. 支持系统托盘。
7. 提供基础 IPC 通信。

## 3. 窗口类型

### 3.1 宠物窗口

用途：

```text
常驻桌面
-> 展示宠物
-> 显示气泡
-> 作为功能入口
```

建议特性：

| 属性 | 建议 |
| --- | --- |
| frame | false |
| transparent | true |
| alwaysOnTop | true |
| resizable | false |
| width | 220 |
| height | 260 |
| skipTaskbar | true |

### 3.2 主面板窗口

用途：

```text
承载待办、聊天、设置、游戏入口
```

建议特性：

| 属性 | 建议 |
| --- | --- |
| frame | true 或自定义标题栏 |
| transparent | false |
| alwaysOnTop | false |
| width | 420 |
| height | 620 |

### 3.3 小游戏窗口

MVP 可以先复用主面板区域，也可以创建独立窗口。

建议先复用主面板，降低窗口管理复杂度。等体验稳定后再拆独立游戏窗口。

## 4. 主进程职责

1. 初始化应用。
2. 创建窗口。
3. 管理窗口显示、隐藏、关闭。
4. 管理托盘菜单。
5. 注册 IPC 接口。
6. 调用本地存储模块完成数据库初始化。
7. 保存窗口位置。

应用壳不定义数据库表结构，不实现任务、宠物、游戏、情绪等业务规则。

## 5. 渲染进程职责

1. 渲染 React 页面。
2. 展示宠物、面板和模块 UI。
3. 通过 preload 暴露的安全 API 调用 IPC。
4. 不直接访问 Node 原生能力。

## 6. IPC 基础接口

```ts
window.appApi.window.showMainPanel()
window.appApi.window.hideMainPanel()
window.appApi.window.toggleMainPanel()
window.appApi.window.openPanel(route)
window.appApi.window.setPetPosition(x, y)
window.appApi.window.getPetPosition()
```

`route` 只能是共享契约中定义的 `PanelRoute`，例如 `tasks`、`chat`、`game`、`settings`。

## 7. 首次开发建议

第一轮只实现：

```text
Electron 启动
-> 显示宠物窗口
-> 宠物窗口中渲染一个占位宠物
-> 点击占位宠物打开主面板
-> 托盘菜单可退出应用
```

不要在本模块第一轮加入：

1. MediaPipe。
2. Phaser。
3. SQLite 复杂表结构。
4. AI 对话。
5. 多宠物资源。

## 8. 验收标准

1. 应用可通过 `npm run dev` 启动。
2. 桌面出现透明宠物窗口。
3. 宠物窗口可拖动。
4. 点击宠物能打开主面板。
5. 主面板关闭后宠物仍然存在。
6. 托盘可以显示应用菜单并退出。
7. DevTools 不报关键错误。
