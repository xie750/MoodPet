# 待办任务模块设计文档

## 1. 模块定位

待办任务模块不是普通任务列表，而是“陪伴式任务管理”。

它的目标不是让用户感到被催促，而是让桌宠陪用户温和地完成今天的事情。

## 2. MVP 功能范围

MVP 只做今日任务：

1. 新建任务。
2. 编辑任务标题。
3. 删除任务。
4. 标记完成。
5. 显示今日任务列表。
6. 完成任务后发布 `task_completed` 事件。
7. 提供任务摘要给状态决策和对话模块使用。

暂不做：

1. 复杂日历。
2. 子任务树。
3. 团队协作。
4. 多端同步。
5. 番茄钟完整系统。

## 3. 数据结构

```ts
type TaskStatus =
  | "todo"
  | "doing"
  | "done"
  | "delayed"
  | "abandoned";

type TaskPriority = "low" | "medium" | "high";

type Task = {
  id: string;
  title: string;
  description?: string;
  status: TaskStatus;
  priority: TaskPriority;
  dueAt?: number;
  completedAt?: number;
  createdAt: number;
  updatedAt: number;
};
```

## 4. 核心流程

### 4.1 创建任务

```text
用户输入任务
-> 保存到本地
-> 触发 task_created
-> 应用编排层决定是否让宠物显示轻提示
```

文案示例：

```text
好，我们一起慢慢来。
```

### 4.2 完成任务

```text
用户点击完成
-> 更新任务状态 done
-> 写入 completedAt
-> 触发 task_completed
-> 应用编排层生成 PetCommand
-> 宠物 celebrate，亲密度 +1
```

文案示例：

```text
完成一个啦，真不错。
```

## 5. 与桌宠联动

待办模块不直接调用桌宠模块。

| 行为 | 宠物反馈 |
| --- | --- |
| 创建任务 | 陪伴提示 |
| 完成任务 | 庆祝动画 |
| 今日全部完成 | 强庆祝 |
| 任务很多且用户 tired | 建议先完成最小任务或休息 |
| 删除任务 | 不惩罚，不负反馈 |

以上反馈由 `task_created`、`task_completed` 等事件触发，具体宠物状态和奖励由应用编排层或状态决策引擎决定。

## 6. 模块边界

待办模块可以做：

1. 管理任务增删改查。
2. 计算 `TaskSummary`。
3. 发布任务领域事件。

待办模块不可以做：

1. 直接修改宠物状态。
2. 直接增加宠物亲密度。
3. 直接读取摄像头状态。
4. 直接决定是否邀请小游戏。

## 7. 任务页面信息架构

建议主面板中任务页包含：

1. 今日任务标题。
2. 输入框。
3. 未完成列表。
4. 已完成列表。
5. 今日完成数量。

不需要复杂统计图。

## 8. IPC 接口

```ts
window.appApi.tasks.listToday()
window.appApi.tasks.create(input)
window.appApi.tasks.update(taskId, patch)
window.appApi.tasks.complete(taskId)
window.appApi.tasks.delete(taskId)
window.appApi.tasks.getSummary()
```

## 9. 验收标准

1. 可以创建任务。
2. 可以完成任务。
3. 可以删除任务。
4. 刷新或重启后任务仍存在。
5. 完成任务后发布 task_completed 事件。
6. 任务模块本身不直接修改宠物亲密度。
7. 删除任务不触发负面反馈。
